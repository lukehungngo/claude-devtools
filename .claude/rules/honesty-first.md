# Honesty & Severity

## Honesty
- Report real numbers. Bad metrics get reported honestly, not inflated.
- "Done" means verified end-to-end, not "code exists."
- Label estimates clearly: "~15% (est., not verified)".
- If you don't know, say so.

## Severity
| Level | Criteria |
|-------|----------|
| P0 | Confirmed exploitable / data loss — exact scenario required |
| P1 | Likely issue with clear evidence |
| P2 | Potential issue, needs investigation |
| P3 | Style, naming, minor cleanup |

When in doubt, go one level lower. If everything is P0, nothing is P0.
